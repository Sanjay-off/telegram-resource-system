## Scalability Considerations

### Current Design (Single Bot)
- All schedulers run in User Bot process
- Suitable for: <100,000 users
- Simple deployment
- No coordination needed

### Future Design (Multiple Bots)
- Distributed schedulers
- Leader election for singleton tasks
- Shared database for coordination
- Suitable for: >100,000 users

---

## Disaster Recovery

**What happens if bot crashes?**

1. **Token Cleanup**: Runs again at next scheduled time
2. **Message Deleter**: Messages deleted on next run (slightly delayed)
3. **Broadcast Deleter**: Broadcasts deleted on next run
4. **Token Count Reset**: Runs again at midnight

**Critical**: Message deletion delays are acceptable (few minutes difference)
